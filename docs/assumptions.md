# 가정

1. 현재 저장소는 Git으로 관리되므로, 큰 단계가 끝날 때마다 일반 커밋으로 체크포인트를 남긴다.
2. LAN 접속 URL은 서버가 감지한 비내부망 IPv4 주소 중 첫 번째 사설 주소를 기본값으로 사용하고, 감지 실패 시 현재 origin 또는 `localhost`를 대체값으로 사용한다.
3. 호스트 전용 권한 복구를 위해 서버가 발급한 별도 `hostToken`을 브라우저 `localStorage`에 저장하고, URL에는 포함하지 않는다.
4. 조사/처형/특수 선거의 타깃은 살아있는 플레이어만 허용한다. PDF에 명시되지 않은 자기 자신 대상 허용 여부는 UX와 악용 방지를 위해 기본적으로 불허 방향으로 구현하되, 세부 구현 전 PDF 재대조 후 필요 시 수정한다.
5. JSON snapshot 영속성은 단일 호스트 PC 환경을 전제로 하며, 원자적 임시 파일 쓰기 후 rename 방식으로 스냅샷 손상 가능성을 낮춘다.
6. 정책 선정(Policy Peek) 시 덱 상단 3장을 볼 수 있어야 하므로, 공개되지 않은 폐기 더미와 섞어 덱을 재구성해야만 3장을 충족할 수 있는 극단 상황이 생기면 입법회의 셔플 규칙과 동일한 방식으로 보충한다.
